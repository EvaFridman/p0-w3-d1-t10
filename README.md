# p0-w3-d1-t10 · Стоимость подписки

> Тип: синтез тем дня (побольше и посложнее) · Репозиторий: один отдельный

> Подключить файл, отсмотреть в браузере.

## Задача

Посчитать, сколько пользователь заплатит за подписку, исходя из выбранного тарифа и его
данных. Это цельный расчёт, где вместе работают почти все темы дня: разбор тарифа через
`switch`, проверка типов входных данных, скидки на ветвлениях и тернарнике, значения по
умолчанию через `??`. Функций ещё не было — поэтому вводные задаёшь переменными сверху файла,
а расчёт ведёшь по шагам, выводя промежуточные значения. Завтра (когда будут функции) этот же
расчёт можно будет завернуть в функцию — сегодня делаем «в лоб».

```
вводные (переменные сверху файла)
  тариф · период · возраст · промокод
          │  switch / ветвления / ?? / тернарник
          ▼
  базовая цена → скидки → итог (по шагам в консоль)
```

## Вводные

В начале `pricing.js` задать переменными:

- `plan` — тариф: `'basic'`, `'pro'` или `'team'`;
- `period` — `'month'` или `'year'`;
- `age` — число (для скидок);
- `promo` — промокод, может отсутствовать.

## Правила расчёта

- базовая цена за месяц зависит от тарифа — определить через `switch` по `plan`
  (basic/pro/team — свои цены; неизвестный тариф → понятное сообщение, а не молчаливый сбой);
- если `period === 'year'` — взять цену за 12 месяцев со скидкой (например −20%);
- возрастная скидка: студенту (условно `age < 23`) или пожилому (`age >= 65`) — например
  −10%; задать через тернарник или ветвление; если `age` не число — скидку не давать и
  отметить это, а не падать;
- промокод: если не передан — `'без промокода'` через `??`; если передан конкретный код
  (например `'WELCOME'`) — ещё −5%;
- итог не может быть отрицательным; вывести расчёт по шагам (база → период → скидки → итог).

## Критерии приёмки

* Для нескольких разных наборов вводных считается корректная итоговая цена, виден ход расчёта
  по шагам.
* Тариф разбирается `switch`'ем с обработкой неизвестного значения; нечисловой `age` обработан
  явной проверкой типа (скидки нет, но программа не падает и не выдаёт `NaN`).
* Значение по умолчанию для промокода — через `??`; ты можешь объяснить, почему не `||`.
* Итоговая цена не бывает отрицательной и не превращается в `NaN` на «кривых» вводных.
